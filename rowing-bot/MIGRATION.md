# Руководство по миграции

Этот документ описывает изменения, внесенные в проект при рефакторинге.

## Основные изменения

### 1. Модульная структура

Проект был разбит на модули:

- **config/** - конфигурация и константы
- **handlers/** - обработчики команд и сообщений
- **keyboards/** - функции создания клавиатур
- **middleware/** - rate limiting и валидация
- **services/** - бизнес-логика

### 2. Улучшенное логирование

- Структурированное логирование с контекстом (chatId, username, и т.д.)
- Логи сохраняются в директории `logs/`
- Разделение на error.log и combined.log
- Поддержка разных уровней логирования через переменную окружения `LOG_LEVEL`

### 3. Мониторинг

- Добавлены метрики Prometheus через `prom-client`
- Endpoint `/metrics` для экспорта метрик
- Метрики включают:
  - Количество сообщений и команд
  - Количество расчетов и их длительность
  - Количество активных пользователей и сессий
  - Количество ошибок и срабатываний rate limit

### 4. Тестирование

- Добавлены unit-тесты с использованием Jest
- Тесты для критических функций:
  - Парсинг времени
  - Расчет модельного времени
  - Rate limiting
  - Валидация данных

### 5. Обновленные зависимости

- `prom-client` - для мониторинга
- `jest` - для тестирования (dev dependency)
- Обновлены версии существующих зависимостей

## Что нужно сделать после миграции

1. Установить новые зависимости:
```bash
npm install
```

2. Создать директорию для логов (создается автоматически, но можно создать вручную):
```bash
mkdir logs
```

3. Обновить переменные окружения (опционально):
```env
LOG_LEVEL=info  # или debug, warn, error
```

4. Запустить тесты для проверки:
```bash
npm test
```

## Обратная совместимость

Все существующие функции сохранены:
- Все команды работают как раньше
- Формат данных не изменился
- API бота не изменился

## Новые возможности

1. **Метрики** - доступны на `/metrics` endpoint
2. **Улучшенное логирование** - более детальная информация в логах
3. **Тесты** - возможность автоматического тестирования
4. **Модульность** - проще добавлять новые функции и исправлять ошибки
